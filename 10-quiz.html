Module 10 Quiz

Streaming

🧠 Multiple Choice (5 questions · pass = 4/5)

Q1. What is TTFT (Time to First Token) and why does it matter for user experience?

TTFT is how long users wait before seeing anything. Streaming drastically reduces TTFT — users see text within milliseconds instead of waiting for the full response.

Q2. What does agent.stream() yield?

stream() yields state dict snapshots. Each yield is a completed node in the agent graph — an LLM response or tool result. Not token-by-token text.

Q3. How do you stream token-by-token text from an agent?

astream_events() fires granular events including on_chat_model_stream which yields each token chunk as it arrives from the LLM.

Q4. What HTTP header prevents Nginx from buffering SSE responses?

X-Accel-Buffering: no tells Nginx not to buffer the response — critical for SSE. Without it, Nginx waits to accumulate a buffer before sending, breaking real-time streaming.

Q5. Which FastAPI response class enables Server-Sent Events?

StreamingResponse(generator, media_type='text/event-stream') enables SSE. Set headers={'X-Accel-Buffering': 'no', 'Cache-Control': 'no-cache'}.

🎤 Interview Questions
Explain the difference between agent.stream() and agent.astream_events() and when to use each.▼
stream() yields state snapshots at the graph node level — one yield per completed step. Use it when you want to show tool call progress or step-by-step reasoning. astream_events() fires fine-grained events: on_chat_model_stream (individual tokens), on_tool_start, on_tool_end, on_chain_start, etc. Use astream_events when you need token-by-token text streaming for the frontend typing effect.
How would you implement streaming in a Next.js frontend consuming a FastAPI SSE endpoint?▼
Frontend: const eventSource = new EventSource('/api/agent/stream?message=...'); eventSource.onmessage = (e) => { const data = JSON.parse(e.data); setTokens(prev => prev + data.token); }; eventSource.onerror = () => eventSource.close();. Backend: yield f'data: {json.dumps({"token": chunk.content})} ' inside StreamingResponse. Include Connection: keep-alive and X-Accel-Buffering: no headers.
What is backpressure in streaming and how do you handle it?▼
Backpressure occurs when the producer (LLM/agent) generates data faster than the consumer (frontend/network) can process it. Handle it with: (1) async generators with await asyncio.sleep(0) to yield control, (2) buffered queues with asyncio.Queue to decouple producer and consumer speeds, (3) client-side: process events in requestAnimationFrame to avoid UI jank, (4) abort the stream if the client disconnects by catching GeneratorExit in the async generator.
