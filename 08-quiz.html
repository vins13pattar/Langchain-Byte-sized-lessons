Module 08 Quiz

Structured Output

🧠 Multiple Choice (5 questions · pass = 4/5)

Q1. What method enables structured output on a chat model?

with_structured_output() wraps the model so its output is automatically parsed into your Pydantic model or TypedDict.

Q2. What is the recommended way to define the output schema?

Pydantic BaseModel is recommended — it provides field descriptions (used in the schema), validation, and type coercion. TypedDict works but has no validation.

Q3. How do you extract a list of entities using structured output?

class ExtractedEntities(BaseModel): entities: list[Entity] — wrap the list in a root model. with_structured_output(ExtractedEntities) returns the whole list in one call.

Q4. What happens when the model generates invalid structured output?

If the model output doesn't match the schema, Pydantic validation fails and raises OutputParserException. Use with_retry() or include_raw=True to handle this gracefully.

Q5. What does include_raw=True add to structured output?

include_raw=True returns {'parsed': MyModel(...), 'raw': AIMessage(...), 'parsing_error': None} — useful for debugging and handling parse failures gracefully.

🎤 Interview Questions
When should you use structured output vs. prompting the model to 'respond in JSON'?▼
Always prefer with_structured_output() in production. Prompting for JSON is fragile — the model may include markdown fences, trailing commas, or extra text that breaks parsing. with_structured_output() uses function-calling (guaranteed valid JSON structure) and validates with Pydantic. The only reason to use prompt-based JSON is if your model doesn't support tool calling.
How would you handle partial structured output when the model only fills some fields?▼
Define optional fields with Optional[type] = None in your Pydantic model. Use include_raw=True to catch validation errors and inspect what was returned. You can also use model_config = ConfigDict(extra='ignore') to allow extra fields. For critical fields, use validator decorators to raise descriptive errors, then retry with a more specific prompt.
Design a Pydantic schema for extracting action items from a meeting transcript.▼
class ActionItem(BaseModel): owner: str = Field(description='Person responsible'); task: str = Field(description='What needs to be done'); deadline: Optional[str] = Field(None, description='Deadline if mentioned'); priority: Literal['high','medium','low'] = 'medium'. class MeetingExtractions(BaseModel): action_items: list[ActionItem]; decisions: list[str]; open_questions: list[str]. Use with_structured_output(MeetingExtractions).
