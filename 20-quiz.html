Module 20 Quiz

Multi-Agent Systems

🧠 Multiple Choice (5 questions · pass = 4/5)

Q1. What is the supervisor pattern in multi-agent systems?

The supervisor agent receives the user request, decides which subagents to invoke, collects results, and synthesizes a final response.

Q2. How are subagents exposed to a supervisor agent?

Wrap each subagent's invoke call in a @tool decorated function. The supervisor calls subagents exactly like it calls any tool via tool calling.

Q3. How do you run multiple subagents in parallel?

asyncio.gather() runs multiple async coroutines concurrently - perfect for calling multiple independent subagents simultaneously.

Q4. How does context flow from supervisor to subagents?

Context does not flow automatically. The supervisor must explicitly pass config with the same context to each subagent it invokes.

Q5. What is the main challenge when debugging multi-agent systems?

Multi-agent traces span multiple agents each with their own LLM calls. LangSmith's trace visualization is essential for understanding the full execution tree.

🎤 Interview Questions
Design a multi-agent research system that searches the web, analyzes data, and writes a report.▼
Agents: (1) SearchAgent: tools=[web_search, scrape_page]. (2) AnalysisAgent: tools=[python_repl, data_viz]. (3) WritingAgent: tools=[write_section, format_markdown]. Supervisor: Route query to SearchAgent. Pass results to AnalysisAgent. Pass analysis to WritingAgent. Use asyncio.gather for search and analysis when they can run in parallel. Thread all agents with the same thread_id for shared context.
What are the failure modes unique to multi-agent systems vs. single agents?▼
(1) Cascading failures: one subagent's error causes the supervisor to make wrong decisions. (2) Context loss: supervisor passes incomplete context to subagents. (3) Cost explosion: supervisor calls expensive agents repeatedly due to poor routing logic. (4) Circular calls: A calls B calls A. (5) Inconsistent state: parallel subagents modify shared state concurrently. Mitigations: structured outputs with error fields, explicit context passing, token budgets, circuit breakers.
How does LangSmith help debug multi-agent systems?▼
LangSmith captures the entire agent graph as a hierarchical trace. You see: which supervisor decision triggered which subagent, full input/output of each agent, token usage and latency per agent, which tool calls happened inside each subagent, and where decisions went wrong. Without LangSmith, debugging multi-agent systems means parsing interleaved logs from multiple agents - extremely difficult.
