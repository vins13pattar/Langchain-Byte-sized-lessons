Module 24 Quiz

Agentic RAG

🧠 Multiple Choice (5 questions · pass = 4/5)

Q1. What distinguishes agentic RAG from standard RAG?

Standard RAG always retrieves with the same query. Agentic RAG uses an LLM to decide: should I retrieve? What query? Is the result good enough? Should I search again?

Q2. What is Corrective RAG (CRAG)?

CRAG adds a relevance grade step: if retrieved docs score below a threshold, the agent falls back to web search - preventing hallucinations from irrelevant context.

Q3. What class is used for relevance grading in agentic RAG?

class RelevanceGrade(BaseModel): grade: Literal['relevant', 'not_relevant']. Use with_structured_output(RelevanceGrade) on a model to grade each retrieved document.

Q4. What is the citation risk in RAG systems?

LLMs sometimes generate plausible-sounding but fabricated source citations. Always verify cited sources exist and match the content before presenting to users.

Q5. What is 'self-RAG'?

Self-RAG interleaves generation and retrieval at the sentence level - the model generates a retrieve token when uncertain, retrieves, then continues generating.

🎤 Interview Questions
Design an agentic RAG pipeline that handles both specific factual queries and broad research questions.▼
Router: LLM classifies query as 'factual' (needs precise retrieval) or 'research' (needs multiple sources). Factual path: vector retrieval, relevance grade, if not_relevant then web search, answer. Research path: query decomposition into 3-5 sub-questions, parallel retrieval for each, aggregate and deduplicate, synthesize. Both paths: output includes source citations with verified URLs. Each path exposed as a tool to the supervisor.
How does CRAG prevent hallucinations from irrelevant context?▼
Standard RAG always injects retrieved docs even if irrelevant - the LLM may hallucinate based on tangentially related content. CRAG: (1) Grade each retrieved doc for relevance. (2) If all docs grade 'not_relevant': discard and fall back to web search. (3) If some grade 'relevant': use only those. The LLM receives only validated relevant context - dramatically reducing hallucination from noise injection.
What are the latency trade-offs of agentic RAG vs. standard RAG?▼
Standard RAG: 1 retrieval + 1 LLM call. Fast (~1s). Agentic RAG: 1 retrieval + 1 grade call + possibly 1 re-retrieval + 1 final LLM call. Slower (~2-4s). Research queries with decomposition: many cycles (~5-10s). Optimization: parallelize sub-question retrieval with asyncio.gather, use fast model for grading, cache embeddings, set max_iterations limit. Show progress ('Searching... Evaluating sources...') to make wait time feel productive.
