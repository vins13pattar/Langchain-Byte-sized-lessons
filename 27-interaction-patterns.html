Interaction Patterns — Feedback, Uploads & HITL UI

⏱ ~18 min💻 Runnable code🖱️ UX patterns

What you'll learn: Five interaction patterns that make agent UIs professional — thumbs up/down feedback collection, file upload with context injection, stop-generation, Human-in-the-Loop approval UI, and suggested follow-up prompts.

🎛️
Analogy: the cockpit vs. the car

A car has a steering wheel and pedals — minimal controls for minimal complexity. A cockpit has hundreds of controls because pilots need fine-grained command over a complex system. Agent UIs are cockpits: users need to stop, steer, correct, upload context, approve actions, and provide feedback. The right interaction patterns make the complexity manageable.

Five Key Interaction Patterns

👍 Feedback Collection

Thumbs up/down per message — drives RLHF and quality monitoring. Store in your DB linked to thread_id + message_id.

📎 File Upload

Attach documents the agent can read — sent to the backend, uploaded to storage, URL injected into the agent's message.

⏹️ Stop Generation

Abort button cancels the streaming fetch. Critical UX — users expect to be able to stop a long response mid-stream.

✅ HITL Approval UI

When agent pauses for approval, show the pending action and Approve/Reject buttons. Resume or redirect based on decision.

💡 Suggested Prompts

Show 2–4 follow-up suggestions after each response. Reduces blank-slate anxiety and guides users to valuable actions.

📋 Copy & Share

Copy individual messages or the full thread as markdown. Share links generate a read-only view of the conversation.
