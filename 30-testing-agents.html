Testing Agents — Unit, Integration & Eval-Based Tests

⏱ ~20 min💻 Runnable code🧪 Testing

What you'll learn: The three-tier testing pyramid for agents — unit tests with mocked LLMs (fast, free), integration tests with real LLMs (slower, catches real failures), and eval-based tests with LangSmith (catches quality regressions). How to write each and when to use them.

🏗️
Analogy: building inspection tiers

You don't hire a building inspector to check every nail — you use different checks at different stages. Framing inspection (fast, cheap) catches big structural problems early. Mechanical/electrical inspection (slower, needs an expert) catches system integration issues. Final occupancy inspection (rare, thorough) certifies the whole thing works. Agent testing is the same: mock tests catch logic errors, integration tests catch model+tool wiring, evals catch quality drift.

🔬 Unit Tests

⚡ ~milliseconds

Mock the LLM. Test tool logic, middleware, routing, state transitions. Zero API calls, zero cost. Runs in CI on every commit.

🔗 Integration Tests

⏱ ~5–30 seconds

Real LLM, real tools. Test the agent end-to-end on a fixed input. Catches prompt breakage. Run on PR merges or nightly.

📊 Eval Tests

🐢 ~minutes

Run agent against a dataset of 50+ examples. Score with LLM-as-judge. Catches quality regressions. Run before major releases.

Unit Tests — Mocking the LLM

pythontests/test_tools.py
import pytest
from unittest.mock import MagicMock, patch
from langchain_core.messages import AIMessage, HumanMessage, ToolMessage

# 1. Test tool logic directly — no LLM involved
from my_agent.tools import search_database

def test_search_database_found():
    result = search_database("product_id: 12345")
    assert "price" in result
    assert "availability" in result

def test_search_database_empty_query():
    with pytest.raises(ValueError, match="Query cannot be empty"):
        search_database("")

# 2. Test the agent with a fake LLM response
from langchain_core.language_models.fake import FakeListChatModel
from my_agent.agent import create_support_agent

def test_agent_calls_correct_tool():
    """Check the agent routes 'find order' to the order_lookup tool."""

    # FakeListChatModel returns pre-scripted responses in order
    fake_responses = [
        AIMessage(
            content="",
            tool_calls=[{
                "name": "order_lookup",
                "args": {"order_id": "ORD-123"},
                "id": "call_abc",
                "type": "tool_call"
            }]
        ),
        AIMessage(content="Your order ORD-123 is shipped!")
    ]

    fake_llm = FakeListChatModel(responses=fake_responses)
    agent = create_support_agent(model=fake_llm)

    result = agent.invoke({"messages": [HumanMessage("Where is order ORD-123?")]})

    messages = result["messages"]
    # Check a tool call happened
    tool_calls = [m for m in messages if hasattr(m, "tool_calls") and m.tool_calls]
    assert len(tool_calls) == 1
    assert tool_calls[0].tool_calls[0]["name"] == "order_lookup"

# 3. Test middleware logic
from my_agent.middleware import RateLimitMiddleware

@pytest.mark.asyncio
async def test_rate_limit_blocks_after_threshold():
    middleware = RateLimitMiddleware(max_calls_per_minute=3)
    user_id = "user_123"

    # First 3 calls should pass
    for _ in range(3):
        assert await middleware.check_rate_limit(user_id) == True

    # 4th call should be blocked
    assert await middleware.check_rate_limit(user_id) == False

Integration Tests — Real LLM, Fixed Inputs

pythontests/test_integration.py
import pytest
import os

# Skip if no API key (don't fail CI when key not set)
pytestmark = pytest.mark.skipif(
    not os.getenv("ANTHROPIC_API_KEY"),
    reason="Integration tests require ANTHROPIC_API_KEY"
)

from my_agent.agent import create_support_agent

@pytest.fixture(scope="module")
def agent():
    """One agent instance per test module — cheaper than per-test."""
    return create_support_agent()

def test_greeting_response(agent):
    """Agent responds sensibly to a greeting."""
    result = agent.invoke({"messages": [("user", "Hello!")]})
    response = result["messages"][-1].content.lower()
    assert any(w in response for w in ["hello", "hi", "help", "assist"])

def test_order_lookup_uses_tool(agent):
    """Agent calls order_lookup tool when asked about an order."""
    result = agent.invoke({"messages": [("user", "Check order ORD-999")]})
    messages = result["messages"]
    # At least one ToolMessage should be present
    tool_messages = [m for m in messages if isinstance(m, ToolMessage)]
    assert len(tool_messages) >= 1

def test_agent_respects_guardrail(agent):
    """Agent declines off-topic requests."""
    result = agent.invoke({"messages": [("user", "Write me a poem about cats")]})
    response = result["messages"][-1].content.lower()
    assert any(w in response for w in ["support", "assist", "focus", "help with"])

@pytest.mark.asyncio
async def test_streaming_returns_tokens(agent):
    """Streaming mode actually yields partial chunks."""
    chunks = []
    async for event in agent.astream_events(
        {"messages": [("user", "What's your return policy?")]},
        version="v2"
    ):
        if event["event"] == "on_chat_model_stream":
            chunks.append(event["data"]["chunk"].content)

    # Should have multiple chunks, not just one big response
    assert len(chunks) > 3

Eval-Based Tests — Quality Regression Prevention

pythontests/test_evals.py
"""
Eval tests — run with: pytest tests/test_evals.py -m eval
These are slow and require both ANTHROPIC_API_KEY and LANGCHAIN_API_KEY.
Run in CI nightly or before releases, not on every commit.
"""
import pytest
from langsmith import evaluate, Client
from langsmith.evaluation import LangChainStringEvaluator
from langchain_openai import ChatOpenAI
from my_agent.agent import create_support_agent

pytestmark = [
    pytest.mark.eval,
    pytest.mark.skipif(
        not os.getenv("LANGCHAIN_API_KEY"),
        reason="Evals require LANGCHAIN_API_KEY"
    )
]

QUALITY_THRESHOLD = 0.80  # Fail if average correctness drops below 80%

def test_support_agent_quality():
    agent = create_support_agent()

    def run_agent(inputs: dict) -> dict:
        result = agent.invoke({"messages": [("user", inputs["question"])]})
        return {"answer": result["messages"][-1].content}

    correctness = LangChainStringEvaluator(
        "labeled_criteria",
        config={"criteria": "correctness", "llm": ChatOpenAI(model="gpt-4o")}
    )

    results = evaluate(
        run_agent,
        data="support-agent-golden-set",
        evaluators=[correctness],
        experiment_prefix="pytest-eval",
        max_concurrency=4,
    )

    df = results.to_pandas()
    avg_score = df["correctness"].mean()

    print(f"\nEval results: avg correctness = {avg_score:.2f}")
    assert avg_score >= QUALITY_THRESHOLD, (
        f"Agent quality dropped below threshold! "
        f"Got {avg_score:.2f}, expected >= {QUALITY_THRESHOLD}"
    )
⚠️
Gotcha: non-determinism makes assertions brittle

LLMs are probabilistic — the same prompt won't always produce the exact same output. Don't assert assert response == "exact string". Instead: check for key words, check tool call names (deterministic), check response length range, or use LLM-as-judge for quality. Also: integration tests can be flaky on model outages. Use pytest-retry for resilience and separate "fast" from "slow" tests using markers.

🛡️ Preventing Prompt Injection in CI

A security-conscious team adds a unit test that checks their InputGuardrailMiddleware blocks known injection patterns: "ignore previous instructions", "system:", and "jailbreak". This runs in 20ms on every commit, with zero API calls. A separate nightly eval test measures whether the guardrails cause excessive false positives on legitimate customer messages. The unit tests catch code changes; the evals catch prompt drift. Together they provide defense-in-depth without slowing down the development cycle.

✅
Key Takeaways

Test agents at three tiers: unit (mocked LLM, fast, every commit), integration (real LLM, fixed inputs, on merge), eval (dataset + scoring, before releases). Use FakeListChatModel for deterministic unit tests. Mark integration and eval tests with pytest.mark.skipif to not block CI when keys are absent. Never assert exact LLM output strings — check structure, key words, or tool call names. Use LLM-as-judge (via LangSmith) for quality scoring, not hand-written string checks.
