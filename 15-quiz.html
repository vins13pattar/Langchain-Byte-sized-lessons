Module 15 Quiz

Custom Middleware

🧠 Multiple Choice (5 questions · pass = 4/5)

Q1. How do you create custom middleware?

Subclass AgentMiddleware and override the lifecycle methods you need: before_model, after_model, wrap_tool_call, dynamic_prompt.

Q2. How does custom middleware access runtime context (e.g., user_id)?

self._context is populated from config['configurable'] passed to agent.invoke(). Access user_id as self._context.get('user_id').

Q3. What does @after_model receive?

@after_model receives the AIMessage returned by the LLM. Use it for output validation, guardrail checks, or logging the model's response.

Q4. In @wrap_tool_call, what does 'yield' do?

yield in @wrap_tool_call is the tool execution point — code before yield runs pre-tool; tool_output = yield runs the tool and captures its result; code after runs post-tool.

Q5. How do you skip middleware for specific tools?

In wrap_tool_call, check the tool_call.function.name: if tool_name in SKIP_LIST: yield; return. Else run your pre/post logic around yield.

🎤 Interview Questions
Write the structure of a custom PII-scrubbing middleware.▼
class PIIScrubber(AgentMiddleware): PATTERNS = [re.compile(r'\b\d{3}-\d{2}-\d{4}\b'), re.compile(r'\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b')]. async def before_model(self, messages, **kwargs): return [scrub(m) for m in messages]. def scrub(self, msg): text = msg.content; [text := p.sub('[REDACTED]', text) for p in self.PATTERNS]; return msg.__class__(content=text). Apply before other middleware so all downstream sees scrubbed content.
How would you implement middleware that A/B tests two different system prompts?▼
class ABTestMiddleware(AgentMiddleware): async def dynamic_prompt(self, prompt, **kw): user_id = self._context.get('user_id', ''); bucket = hash(user_id) % 2; if bucket == 0: return PROMPT_A; return PROMPT_B. Log which prompt was used alongside the outcome in @after_model. After enough samples, compare correctness/satisfaction scores between buckets to determine the winner. This gives you statistically rigorous prompt comparison in production.
What are the failure modes of custom middleware and how do you defend against them?▼
(1) Exception in before_model → agent never calls LLM. Wrap in try/except, log and return original messages on failure. (2) Exception in after_model → agent response is lost. Same pattern — fail open, return original response. (3) Exception in wrap_tool_call before yield → tool never runs. Catch and let tool proceed. (4) Slow middleware → increases latency for every request. Profile middleware; run I/O-bound checks (rate limit, logging) with timeouts. (5) Middleware has state → unsafe in multi-threaded environments. Keep middleware stateless; read from self._context.
