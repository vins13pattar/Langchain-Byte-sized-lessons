Agent Chat UI — Open-Source Starter Kit

⏱ ~15 min🚀 Getting Started⚡ Next.js + FastAPI

What you'll learn: LangChain's official open-source agent-chat-ui starter — what it includes, how to clone and customize it, how to wire it to your own backend, and what to change first when adapting it to a real product.

🏠
Analogy: a model home

A model home shows you a fully furnished, working house — real floors, real kitchen, real wiring. You don't live in it as-is, but it shows you exactly how everything fits together and gives you something concrete to start from. agent-chat-ui is your model home: a real, deployed, working agent chat app that you clone and adapt rather than build from scratch.

What's in the Starter

The official agent-chat-ui repo (github.com/langchain-ai/agent-chat-ui) ships a production-ready reference implementation. It's the reference app for LangChain's agent stack.

  • Next.js 14 frontend — App Router, TypeScript, Tailwind CSS
  • FastAPI backend — Streaming SSE, JWT auth middleware wired up
  • useAgentStream hook — Fetch + ReadableStream (as covered in Module 26)
  • LangSmith tracing — Pre-configured with env vars; just add your key
  • Thread management — New conversation / continue conversation via thread_id
  • Markdown rendering — react-markdown + remark-gfm + rehype-highlight
  • Tool call display — Collapsible tool call cards with input/output
  • Dark mode — Tailwind dark: variant, system preference detection

Getting Started in 5 Minutes

bashterminal
# Clone the starter
git clone https://github.com/langchain-ai/agent-chat-ui.git
cd agent-chat-ui

# Install frontend dependencies
cd frontend && npm install

# Install backend dependencies
cd ../backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env: add ANTHROPIC_API_KEY, LANGCHAIN_API_KEY, etc.

# Run both servers (frontend on :3000, backend on :8000)
npm run dev  # from /frontend
uvicorn main:app --reload  # from /backend

# Visit http://localhost:3000 — you should see the chat UI

The Three Files You'll Always Customize

  • 1
    backend/agent.py

    Replace the example agent with yours. Change the model, system prompt, tools list, and any middleware. This is the core of your product.

  • 2
    frontend/components/ChatInterface.tsx

    Add your branding: logo, color scheme, welcome message, suggested prompts. This is what users see first.

  • 3
    backend/main.py

    Wire in your real auth: replace the placeholder JWT check with your actual auth provider (Auth0, Supabase, Clerk, etc.).

Replacing the Example Agent

pythonbackend/agent.py — customized
"""Drop in your own agent here. The rest of the stack stays unchanged."""
from langchain_anthropic import ChatAnthropic
from langchain.agents import create_agent
from langchain.tools import tool
from my_company.tools import search_crm, lookup_order, send_notification
from my_company.middleware import AuthMiddleware, RateLimitMiddleware

# 1. Your model of choice
model = ChatAnthropic(model="claude-opus-4-5", temperature=0)

# 2. Your real tools
TOOLS = [search_crm, lookup_order, send_notification]

# 3. Your system prompt
SYSTEM_PROMPT = """You are a customer support agent for Acme Corp.
Help customers with orders, returns, and account questions.
Always be professional and empathetic. If you cannot help,
escalate to a human agent by calling the escalate_to_human tool."""

# 4. Your middleware
MIDDLEWARE = [AuthMiddleware(), RateLimitMiddleware(max_per_minute=20)]

# 5. Create agent — main.py calls create_agent() and imports TOOLS etc.
def get_agent():
    return create_agent(
        model=model,
        tools=TOOLS,
        prompt=SYSTEM_PROMPT,
        middleware=MIDDLEWARE,
    )

CopilotKit: embeddable agent UI as a React component

For embedding a chat UI inside an existing app (not a standalone page), consider CopilotKit (@copilotkit/react-ui) — it wraps LangChain agents in a <CopilotSidebar> or <CopilotPopup> component. Drop it into any React app with one component and connect it to your FastAPI backend via useCoAgent(). Agent Chat UI is better for standalone apps; CopilotKit is better for embedding into an existing product.

⚠️
Gotcha: the Next.js API route is a proxy, not the agent

The frontend never calls your FastAPI backend directly in production — it goes through app/api/chat/route.ts, which proxies the request. This is intentional: the Next.js route attaches auth headers, validates session tokens, and keeps your backend URL private. Don't skip this proxy layer to "simplify" the setup — you'll expose your backend directly to the internet.

🏗️ Internal Ops Tool in a Weekend

A solo developer needs an internal chat UI for their company's ops team — 20 people, no designers available. They clone agent-chat-ui, replace agent.py with their Jira + Confluence + Slack-connected agent, swap the logo, and change the color palette in tailwind.config.ts. Total customization time: 3 hours. They deploy the backend to Railway (free tier) and the frontend to Vercel (free tier). Monday morning, the ops team has a working AI assistant connected to all their internal tools — without building a single UI component from scratch.

✅
Key Takeaways

agent-chat-ui gives you a production-ready chat UI wired to a FastAPI streaming backend — clone, customize, deploy. The three files you always edit: backend/agent.py (your agent), ChatInterface.tsx (your branding), main.py (your auth). The Next.js API route is a security proxy — keep it. For embedding in an existing app, use CopilotKit instead. LangSmith tracing is pre-wired; just add your key.
