Event Streaming with astream_events()

⏱ ~20 min💻 Runnable code🎯 Event filtering

What you'll learn: How astream_events() gives you a structured event bus instead of raw chunks, the key event types and when they fire, how to filter events for specific use cases, and how to emit custom events from inside tools.

📡
Analogy: event bus vs. raw byte stream

stream() is like reading a raw serial port — you get bytes and have to figure out what they mean. astream_events() is like subscribing to a typed message bus: every event has a name, event, and data field. You subscribe to exactly what you care about — model tokens, tool starts, tool ends, custom events — and ignore the rest. Much easier to build a rich UI from.

stream() vs. astream_events() — When to Use Each

Use stream() when you need simplicity: token-by-token output for a basic chat UI, or step-by-step debugging. It has lower overhead.

Use astream_events() when you need granularity: different UI reactions for model start, tool calls, tool results, and custom tool events (like progress updates from a long-running tool). It's the right tool for production chat UIs with multiple visual states.

The Event Lifecycle

Event NameWhen it firesKey data
on_chain_start Agent starts processing input — the messages sent in
on_chat_model_start Model call begins input — messages sent to model
on_chat_model_stream Each token from the model chunk — AIMessageChunk with text or tool_call_chunks
on_chat_model_end Model call completes output — complete AIMessage
on_tool_start Tool execution begins input — tool arguments dict
on_tool_end Tool execution completes output — tool return value (ToolMessage)
on_custom_event Tool emits a custom event name, data — your custom payload
on_chain_end Agent finishes output — final agent state

Basic astream_events() Usage

pythonevent_streaming.py
import asyncio
from langchain.agents import create_agent
from langchain.chat_models import init_chat_model
from langchain.tools import tool
from langchain_core.messages import HumanMessage

@tool
def search_database(query: str) -> str:
    """Search the product database."""
    return f"Found: Widget A, Widget B, Widget C for query '{query}'"

agent = create_agent(
    model=init_chat_model("openai:gpt-4o-mini"),
    tools=[search_database],
)

async def main():
    input_data = {"messages": [HumanMessage("Search for blue widgets")]}

    # astream_events returns structured event dicts
    async for event in agent.astream_events(input_data, version="v2"):
        event_type = event["event"]
        event_name = event.get("name", "")
        data = event.get("data", {})

        if event_type == "on_chat_model_stream":
            # Token-by-token text output
            chunk = data.get("chunk")
            if chunk and hasattr(chunk, "content") and chunk.content:
                print(chunk.content, end="", flush=True)

        elif event_type == "on_tool_start":
            # Tool is about to run
            tool_input = data.get("input", {})
            print(f"\n[🔧 Calling: {event_name}({tool_input})]")

        elif event_type == "on_tool_end":
            # Tool finished
            output = data.get("output")
            print(f"[✅ Tool result: {str(output)[:60]}]")

        elif event_type == "on_chain_end":
            print("\n[Done]")

asyncio.run(main())

Filtering Events by Type and Name

pythonevent_filtering.py
async def filtered_stream(agent, input_data):
    # ── Only text tokens (no tool events) ─────────────────────────────────────
    async for event in agent.astream_events(input_data, version="v2"):
        if event["event"] == "on_chat_model_stream":
            chunk = event["data"].get("chunk")
            if chunk and chunk.content:
                yield chunk.content

    # ── Filter by specific tool name ───────────────────────────────────────────
    async for event in agent.astream_events(input_data, version="v2"):
        if event["event"] == "on_tool_start" and event["name"] == "search_database":
            print(f"Searching: {event['data']['input']}")

    # ── include/exclude filtering via parameters ───────────────────────────────
    # Only events from specific tags (set via runnable.with_config):
    async for event in agent.astream_events(
        input_data,
        version="v2",
        include_names=["search_database", "get_weather"],  # only these tools
        include_types=["on_tool_start", "on_tool_end"],    # only tool events
    ):
        print(event)

    # ── Exclude noisy events ───────────────────────────────────────────────────
    async for event in agent.astream_events(
        input_data,
        version="v2",
        exclude_types=["on_chain_start", "on_chain_end"],  # skip chain wrapping
    ):
        print(f"{event['event']}: {str(event.get('data', ''))[:40]}")

Custom Events from Tools

Long-running tools can emit progress events mid-execution using runtime.stream_writer. These surface as on_custom_event in the event stream.

pythoncustom_events.py
import asyncio
import time
from langchain.tools import tool
from langchain.tools import ToolRuntime
from langchain.agents import create_agent
from langchain.chat_models import init_chat_model
from langchain_core.messages import HumanMessage

@tool
async def analyze_report(report_id: str, runtime: ToolRuntime) -> str:
    """Analyze a large report. Sends progress updates.

    Args:
        report_id: The report to analyze.
    """
    steps = [
        ("Downloading report...", 0),
        ("Parsing content...", 25),
        ("Running analysis...", 50),
        ("Generating insights...", 75),
        ("Finalizing...", 90),
    ]

    for message, progress in steps:
        # Emit a custom event — visible via astream_events as on_custom_event
        runtime.stream_writer({
            "type": "progress",
            "message": message,
            "progress": progress,
        })
        await asyncio.sleep(0.5)  # simulate work

    return f"Report {report_id}: 3 key risks identified, 2 opportunities found."

agent = create_agent(
    model=init_chat_model("openai:gpt-4o-mini"),
    tools=[analyze_report],
)

async def main():
    input_data = {"messages": [HumanMessage("Analyze report Q4-2024")]}

    async for event in agent.astream_events(input_data, version="v2"):
        if event["event"] == "on_custom_event":
            # Progress events from the tool
            data = event["data"]
            if data.get("type") == "progress":
                pct = data["progress"]
                msg = data["message"]
                bar = "█" * (pct // 10) + "░" * (10 - pct // 10)
                print(f"\r[{bar}] {pct}% — {msg}", end="", flush=True)

        elif event["event"] == "on_chat_model_stream":
            chunk = event["data"].get("chunk")
            if chunk and chunk.content:
                print(chunk.content, end="", flush=True)

asyncio.run(main())

Building a Multi-State Chat UI

This pattern drives a rich UI with distinct visual states: thinking indicator → tool call banner → streaming text → done.

pythonui_state_machine.py
from enum import Enum

class UIState(Enum):
    IDLE = "idle"
    THINKING = "thinking"       # model is reasoning
    USING_TOOL = "using_tool"   # tool call in progress
    STREAMING = "streaming"     # final answer tokens flowing
    DONE = "done"

async def drive_ui(agent, message: str):
    """Returns a stream of (UIState, data) tuples."""
    current_state = UIState.IDLE
    input_data = {"messages": [HumanMessage(message)]}

    async for event in agent.astream_events(input_data, version="v2"):
        etype = event["event"]

        if etype == "on_chat_model_start":
            current_state = UIState.THINKING
            yield current_state, {"msg": "Agent is thinking..."}

        elif etype == "on_tool_start":
            current_state = UIState.USING_TOOL
            yield current_state, {
                "tool": event["name"],
                "args": event["data"].get("input", {}),
            }

        elif etype == "on_tool_end":
            # Back to thinking before next model call
            current_state = UIState.THINKING
            yield current_state, {"msg": "Processing results..."}

        elif etype == "on_chat_model_stream":
            chunk = event["data"].get("chunk")
            if chunk and chunk.content and not chunk.tool_call_chunks:
                current_state = UIState.STREAMING
                yield current_state, {"token": chunk.content}

        elif etype == "on_chain_end":
            current_state = UIState.DONE
            yield current_state, {}

# Usage:
async def render():
    async for state, data in drive_ui(agent, "Search for the latest AI news"):
        if state == UIState.THINKING:
            print(f"⏳ {data['msg']}")
        elif state == UIState.USING_TOOL:
            print(f"🔧 Calling {data['tool']} with {data['args']}")
        elif state == UIState.STREAMING:
            print(data["token"], end="", flush=True)
        elif state == UIState.DONE:
            print("\n✅ Done")

import asyncio
asyncio.run(render())
⚠️
Gotcha: always use version="v2"

astream_events() requires a version parameter. Always pass version="v2" — it's the current stable version. The v1 API has different event shapes and is considered legacy. If you omit version, you'll get a warning.

⚠️
Gotcha: astream_events() is async only

There is no synchronous stream_events(). If you're not in an async context, use stream() instead, or run the async version in an event loop with asyncio.run(). In FastAPI/Django async views this is fine natively.

Real-World Use Case

🔬 Research Agent with Live Progress

A research assistant UI has four visual zones: a "Researching..." spinner (fires on on_tool_start for the web search tool), a sidebar showing "Sources found: 3" (fires on on_tool_end, parsing the results count), a progress bar from custom events emitted by a slow PDF parser tool, and a streaming text response zone (fires on on_chat_model_stream). All four zones update independently from the same astream_events() loop — the client filters by event type and routes to the right UI component.

✅
Key Takeaways

Use astream_events(version="v2") when you need typed events, not raw chunks. Key events: on_chat_model_stream (tokens), on_tool_start/end (tool lifecycle), on_custom_event (progress from tools). Filter with include_names, include_types. Emit custom events from tools via runtime.stream_writer().
