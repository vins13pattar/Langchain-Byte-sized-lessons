Module 04 Quiz

Messages

🧠 Multiple Choice (5 questions · pass = 4/5)

Q1. Which message type represents the AI's response?

AIMessage holds the model's response. There is no 'AssistantMessage' class in LangChain — it's always AIMessage.

Q2. What is SystemMessage used for?

SystemMessage sets the agent's role, constraints, and behavior. It's typically the first message and persists throughout the conversation.

Q3. What message type carries tool execution results back to the model?

ToolMessage holds the string output of a tool call. It must include tool_call_id matching the AIMessage's tool call.

Q4. How do you access the text of a HumanMessage?

All LangChain message types expose their text via .content. For multimodal messages, content can be a list of dicts.

Q5. What role does trim_messages() serve?

trim_messages() drops old messages (keeping system + recent) to prevent the conversation from exceeding the model's context window.

🎤 Interview Questions
Why does LangChain use a messages list instead of a simple string for conversation history?▼
A messages list preserves structure: who said what (role), tool call IDs linking AIMessage to ToolMessage, multimodal content (images, files), and metadata. A string would lose all of this, making it impossible to properly handle tool calls, streaming, or multi-turn context management.
Explain the role of ToolMessage and when it appears in a conversation.▼
ToolMessage appears after an AIMessage that contains a tool_calls field. The agent executes the tool, wraps the output in a ToolMessage with the matching tool_call_id, and appends it to the conversation. The model then sees the tool result and decides whether to call another tool or respond directly.
How would you implement message trimming to handle very long conversations?▼
Use trim_messages(messages, max_tokens=4000, token_counter=model, strategy='last', keep_system=True). This keeps the system message, then fills remaining space with the most recent messages. You can also implement a summarization strategy: summarize the oldest messages into a single HumanMessage before trimming.
