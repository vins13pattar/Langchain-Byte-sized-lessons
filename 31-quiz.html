Module 31 Quiz

Agent Chat UI

Q Multiple Choice (5 questions, pass = 4/5)

Q1. What is agent-chat-ui?

agent-chat-ui is LangChain's opinionated starter: Next.js frontend with streaming chat UI, FastAPI backend with agent endpoint, Docker Compose for local dev. Clone and customize.

Q2. What are the three files most commonly customized in agent-chat-ui?

agent.py: your agent logic. ChatInterface.tsx: UI customization. main.py: API endpoints and middleware. Everything else is boilerplate you rarely touch.

Q3. Why is the Next.js API route a security proxy rather than a direct connection?

The Next.js API route is the security layer. It validates the user's session, checks permissions, and forwards requests. FastAPI can be on an internal network with no public exposure.

Q4. What is CopilotKit used for in agent-chat-ui variants?

CopilotKit provides React hooks and components for embedding agents into existing apps: useCoAgent for agent state, CopilotSidebar for chat UI, generative UI for dynamic components.

Q5. What Docker command runs the agent-chat-ui stack locally?

docker-compose up --build starts both the Next.js frontend (port 3000) and FastAPI backend (port 8000). Set env vars in .env before running.

Walk through customizing agent-chat-ui to add a new tool and display its results in the UI.▼
(1) backend/agent.py: add @tool def search_products(query: str): ... to tools list in create_agent(). (2) backend/main.py: ensure tool result event is forwarded in the streaming response. (3) frontend/components/ChatInterface.tsx: in the SSE message handler, check for event.type === 'tool_result' && event.tool === 'search_products' - render a ProductGrid component with event.data. (4) frontend/components/ProductGrid.tsx: display product cards. (5) Test: docker-compose up, ask agent to search, verify products appear as rich UI cards.
How would you add multi-user support with auth to agent-chat-ui?▼
(1) Add NextAuth.js to Next.js: configure provider (Google, GitHub, etc.). (2) Wrap app in SessionProvider. (3) In API route: const session = await getServerSession(); if (!session) return 401. (4) Add user_id to FastAPI request: fetch('/api/agent', {headers: {'Authorization': Bearer token}}). (5) FastAPI: verify token, extract user_id. (6) Pass to agent: config={'configurable': {'thread_id': f'{user_id}-{thread_id}', 'context': {'user_id': user_id}}}. (7) Add AsyncPostgresSaver for persistent per-user memory.
What customizations would you make to agent-chat-ui for an enterprise customer service deployment?▼
UI: custom branding (logo, colors via CSS variables), welcome message with company name, suggested prompts specific to product. Backend: add authentication middleware (SSO/SAML), rate limiting per user, audit logging of all interactions, PII scrubbing middleware. Agent: custom system prompt with company context, tools connected to CRM and knowledge base. Deployment: health check endpoint, Prometheus metrics, structured logging, Docker Compose with Postgres for checkpointer. Remove: demo tools, example data, development-only endpoints.
