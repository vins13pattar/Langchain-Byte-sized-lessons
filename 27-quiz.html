Module 27 Quiz

Interaction Patterns

Q Multiple Choice (5 questions, pass = 4/5)

Q1. How do you implement a thumbs up/down feedback mechanism for agent responses?

POST /api/feedback {message_id, rating: 'up'|'down', comment?}. Store in your DB. LangSmith SDK: client.create_feedback(run_id, key='user_rating', score=1). This builds your eval dataset.

Q2. Where should file validation for uploaded files happen?

Browser validation (file type, size) gives immediate feedback. Server validation is mandatory - browsers can be bypassed. Server: check MIME type from file content (not filename), scan for malware, enforce size limits.

Q3. What are 'suggested prompts' and why do they improve UX?

Suggested prompts (clickable examples) show users what the agent can do and give a starting point. Critical for discovery - most users don't know what to ask without examples.

Q4. How do you implement a human-approval card UI for HITL agents?

ApprovalCard shows: what action the agent wants to take (email to, subject, content), and Approve/Reject buttons. Button click POSTs to /api/approve/{thread_id} which calls agent.update_state + ainvoke.

Q5. What should you display while the agent is using tools?

Show which tool is active. Users tolerate waiting better when they see progress. Use astream_events on_tool_start event to display 'Using [tool name]...' in real time.

A Interview Questions
Design the feedback collection and LangSmith integration for production quality monitoring.▼
Frontend: after each assistant message, show thumbs up/down + optional comment. POST /api/feedback {thread_id, run_id, rating, comment}. Backend: store in feedback table (thread_id, run_id, rating, comment, user_id, timestamp). Sync to LangSmith: client.create_feedback(run_id, key='user_rating', score=1 if rating=='up' else 0, comment=comment). Weekly review: LangSmith Dashboard, filter low-rated runs. Build eval dataset from flagged examples. Track rating trend over time - detect regressions.
How would you implement a multi-step file upload and processing flow with progress updates?▼
(1) Client: upload file to signed S3 URL (not through agent). (2) Client POSTs file metadata {s3_key, filename, size} to /api/process. (3) Backend creates a job_id, enqueues processing job. (4) Client polls GET /api/jobs/{job_id}/status OR subscribes to SSE endpoint. (5) Backend emits: {status: 'validating'}, {status: 'extracting_text'}, {status: 'indexing', progress: 45}, {status: 'done', result_id: ...}. (6) Client shows progress bar from SSE events. (7) On done: trigger agent with result_id as context.
What data should you collect from every agent interaction to enable continuous improvement?▼
Every interaction: thread_id, user_id, message_count, total_latency, total_tokens, tools_called (list), final_answer (first 200 chars hash). User feedback: rating, comment, message_id. Agent failures: exception type, stack trace, last messages. Quality signals: did user immediately rephrase? (negative signal); did conversation continue productively? (positive signal). Weekly: compute metrics - avg latency, P95 latency, tokens/session, tool_error_rate, user_satisfaction_score. Alert on regressions.
