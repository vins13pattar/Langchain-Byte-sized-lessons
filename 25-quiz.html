Module 25 Quiz

Frontend Architecture

Q Multiple Choice (5 questions, pass = 4/5)

Q1. Why should you never run LangChain agents directly in the browser?

Browser code is visible to anyone with DevTools. Running agents client-side would expose API keys (OpenAI, Anthropic), giving attackers unlimited access to your LLM accounts.

Q2. What is the role of the Next.js API route in agent architecture?

The Next.js API route sits between browser and Python backend. It validates auth, enforces rate limits, and proxies requests - the browser never touches the agent directly.

Q3. What HTTP header must your server set to enable real-time SSE streaming?

X-Accel-Buffering: no prevents Nginx from buffering the SSE response. Without it, Nginx waits to accumulate a buffer before sending - breaking real-time streaming.

Q4. What is the recommended pattern for agent state in the frontend?

Frontend UI state (loading, messages, errors) lives in React. Conversation state (messages history) lives in the backend checkpointer. Never mix them.

Q5. Which deployment platforms support SSE without timeout issues?

Vercel serverless functions have a 30s timeout - SSE streams are cut off. Use platforms that support long-lived connections for streaming agents.

Next Quiz
A Interview Questions
Draw the architecture of a production LangChain web app with streaming.▼
Browser (React) <-- SSE --> Next.js API Route (/api/agent) <-- HTTP/SSE --> FastAPI backend <-- LLM API --> Anthropic/OpenAI. Browser: EventSource('/api/agent'), displays tokens as they arrive. Next.js API: validates JWT, checks rate limit, proxies to FastAPI with fetch(). FastAPI: StreamingResponse wrapping agent.astream_events(). FastAPI manages LLM API keys. Browser never sees API keys. Next.js handles auth. FastAPI handles agent logic.
How do you handle authentication in this architecture?▼
Next.js API route: verify JWT from Authorization header (using your auth provider - Auth0, NextAuth, Clerk). Extract user_id from validated token. Pass user_id to FastAPI as a trusted header (X-User-Id: user_id). FastAPI trusts this header only from Next.js (use internal network or shared secret). Pass user_id to agent via config['configurable']['context']['user_id']. Never send raw user data from the browser directly to FastAPI.
What are the trade-offs between WebSockets and SSE for agent streaming?▼
SSE: simpler, HTTP-native, works through all proxies, auto-reconnect built in, one direction (server to client) only. WebSockets: bidirectional, lower overhead for many messages, complex proxy setup, manual reconnect logic. For agents: SSE is usually sufficient - the user sends one message, the agent streams back. Use WebSockets only if you need real-time bidirectional communication (e.g., HITL where the agent asks the user questions mid-stream).
