Rendering Patterns — Streaming, Markdown & Tool Call UI

⏱ ~20 min💻 Runnable code🎨 UI components

What you'll learn: How to consume SSE in React and build smooth token-by-token streaming, render markdown safely, and show tool calls as they happen — the three UI challenges specific to agent applications.

📺
Analogy: live TV vs. recorded shows

A recorded show loads fully before you see anything — that's a non-streaming response. Live TV shows frames as they're broadcast — that's token streaming. Live TV is more engaging even though the content is the same. For AI agents, showing text as it's generated cuts perceived latency by 10x and makes the product feel alive. The rendering challenge is making incomplete text look good at every frame.

Consuming SSE in React

typescripthooks/useAgentStream.ts
import { useState, useCallback, useRef } from 'react'

interface Message {
  id: string
  role: 'user' | 'assistant'
  content: string
  toolCalls?: ToolCall[]
  isStreaming?: boolean
}

interface ToolCall {
  name: string
  status: 'running' | 'done'
  input?: Record
}

export function useAgentStream(threadId: string) {
  const [messages, setMessages] = useState([])
  const [isStreaming, setIsStreaming] = useState(false)
  const abortRef = useRef(null)

  const sendMessage = useCallback(async (userMessage: string) => {
    if (isStreaming) return

    // Add user message immediately (optimistic)
    const userMsg: Message = { id: Date.now().toString(), role: 'user', content: userMessage }
    const assistantId = (Date.now() + 1).toString()
    const assistantMsg: Message = { id: assistantId, role: 'assistant', content: '', isStreaming: true, toolCalls: [] }

    setMessages(prev => [...prev, userMsg, assistantMsg])
    setIsStreaming(true)

    abortRef.current = new AbortController()

    try {
      const res = await fetch('/api/chat/stream', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ message: userMessage, threadId }),
        signal: abortRef.current.signal,
      })

      const reader = res.body!.getReader()
      const decoder = new TextDecoder()
      let buffer = ''

      while (true) {
        const { done, value } = await reader.read()
        if (done) break

        buffer += decoder.decode(value, { stream: true })
        const lines = buffer.split('\n')
        buffer = lines.pop() ?? ''

        for (const line of lines) {
          if (!line.startsWith('data: ')) continue
          const data = JSON.parse(line.slice(6))

          if (data.type === 'token') {
            // Append token to assistant message
            setMessages(prev => prev.map(m =>
              m.id === assistantId ? { ...m, content: m.content + data.content } : m
            ))
          } else if (data.type === 'tool_start') {
            setMessages(prev => prev.map(m =>
              m.id === assistantId
                ? { ...m, toolCalls: [...(m.toolCalls ?? []), { name: data.name, status: 'running', input: data.input }] }
                : m
            ))
          } else if (data.type === 'tool_end') {
            setMessages(prev => prev.map(m =>
              m.id === assistantId
                ? { ...m, toolCalls: (m.toolCalls ?? []).map(tc => tc.name === data.name ? { ...tc, status: 'done' } : tc) }
                : m
            ))
          } else if (data.type === 'done') {
            setMessages(prev => prev.map(m =>
              m.id === assistantId ? { ...m, isStreaming: false } : m
            ))
          }
        }
      }
    } catch (err) {
      if ((err as Error).name !== 'AbortError') console.error(err)
    } finally {
      setIsStreaming(false)
    }
  }, [threadId, isStreaming])

  const stopStreaming = useCallback(() => {
    abortRef.current?.abort()
    setIsStreaming(false)
  }, [])

  return { messages, isStreaming, sendMessage, stopStreaming }
}

function formatToolName(name: string): string {
  return name.replace(/_/g, ' ').replace(/\b\w/g, c => c.toUpperCase())
}
⚠️
Gotcha: never use dangerouslySetInnerHTML for markdown

If you render agent-generated markdown as raw HTML, a malicious prompt injection could produce <script> tags that execute in your users' browsers — an XSS attack. Always use a sanitizing markdown renderer like react-markdown with rehype-sanitize, never raw innerHTML or dangerouslySetInnerHTML.

✅
Key Takeaways

Use the Fetch + ReadableStream API (not EventSource) for SSE — it supports POST and custom headers. Append tokens to the message content on each data.type === 'token' event. Show tool calls as they start (tool_start) and mark them done (tool_end) for transparent UX. Use react-markdown with rehype-sanitize for safe markdown rendering. Add a blinking cursor during streaming by appending a CSS-animated element when isStreaming is true.
